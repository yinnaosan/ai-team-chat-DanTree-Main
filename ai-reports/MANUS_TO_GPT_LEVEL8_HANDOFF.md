## 1. WHAT WAS BUILT

LEVEL8 adds a full persistence and productization layer on top of LEVEL7.1B. All advisory decisions, guard logs, and portfolio snapshots are now written to the database and queryable via tRPC.

### New File
| File | Purpose |
|------|---------|
| `server/portfolioPersistence.ts` | Phase 2: write helpers + Phase 3: snapshot/replay system + query helpers |
| `server/level8.test.ts` | 4 validation test cases |

### DB Schema (5 new tables, migration applied)
| Table | Purpose |
|-------|---------|
| `portfolio` | One row per user (userId FK, name, timestamps) |
| `portfolio_position` | Active positions per ticker (allocationPct, actionLabel, decisionBias, fusionScore, sizingBucket, isActive) |
| `decision_log` | Every decision from every pipeline run (fusionScore, decisionBias, actionLabel, sizingBucket, allocationPct, advisoryOnly=true always) |
| `guard_log` | Every guard event per ticker (dominantGuard, suppressed, decayMultiplier, decayTrace JSON, safetyReport JSON) |
| `portfolio_snapshot` | Full Level7PipelineOutput JSON per run (guardStatus, totalTickers, createdAt) |

### tRPC API (5 endpoints in `portfolioDB` router)
| Endpoint | Type | Description |
|----------|------|-------------|
| `portfolioDB.getMyPortfolio` | query | Returns portfolio + active positions + latest snapshot for logged-in user |
| `portfolioDB.getDecisionLog` | query | Recent decision log (limit 1-100, default 50) |
| `portfolioDB.getGuardLog` | query | Recent guard log (limit 1-100, default 50) |
| `portfolioDB.replayDecision` | query | Replay decision + guard path for a ticker at a given snapshot |
| `portfolioDB.persistRun` | mutation | Persist a Level7 pipeline run; always returns `advisory_only: true` |

**User isolation:** All endpoints use `ctx.user.id` → `getPortfolioByUserId()` → portfolioId chain. No cross-user data leakage possible.
