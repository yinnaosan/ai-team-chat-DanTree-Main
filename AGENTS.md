## SECTION 1 — SYSTEM OVERVIEW

**DanTree** is a decision-support investment research platform. It is structured as a sequential reasoning pipeline:

```
Radar → Candidate → Analysis → Action
```

| Layer | Component | Role |
|---|---|---|
| Discovery | OpportunityRadar | Surface macro/sector/policy signals |
| Intake | CandidatePool | Formal SELECT-stage candidates |
| Research | Level 1–4 Pipeline | Deep multi-layer analysis |
| Decision | DecisionStrip | Final structured decision output |
| Context | CycleEngine | Macro cycle + sector rotation context |

**DanTree is NOT:**
- A trading bot
- A recommendation engine
- A signal generator
- A BUY/SELL alert system

**DanTree IS:**
- A structured reasoning system
- A decision-support layer for human investors
- A pipeline that converts signals into analyzed, contextualized decisions

## SECTION 2 — CORE PRINCIPLES

### Investment State Machine
Every asset in DanTree moves through exactly one path:

```
SELECT → WAIT → BUY → HOLD → SELL
```

No asset skips stages. No asset enters BUY without passing through SELECT and WAIT.

### Foundational Rules

| Rule | Meaning |
|---|---|
| **Logic > Speed** | A slow correct decision beats a fast wrong one |
| **Timing ≠ Decision** | Knowing the right time does not mean the decision is right |
| **Volume confirms, not leads** | Volume validates a move; it does not initiate one |
| **Hidden layer required** | Every analysis must include a non-obvious, hidden insight |
| **Cycle context mandatory** | No decision is valid without macro cycle awareness |

## SECTION 3 — COMPONENT ROLES

### OpportunityRadar
- **Purpose:** Early signal discovery from macro, sector, policy, and theme data
- **Output:** Opportunity cards with WHY (Surface / Trend / Hidden) and confidence score
- **Constraint:** NEVER outputs a BUY signal. Outputs discovery signals only.
- **State:** Feeds CandidatePool via 「加入观察池」 action

### CandidatePool
- **Purpose:** Formal SELECT-stage intake layer
- **Output:** Candidate list with lifecycle status (SELECT / WATCH / PROMOTED / PASS)
- **Constraint:** Candidates are NOT buy recommendations. They are research candidates.
- **State:** Feeds Level 1–4 pipeline via 「开始研究」 action → auto-promotes to PROMOTED

### Level 1–4 Analysis Pipeline
- **Level 1:** Macro + fundamental overview (FRED, SimFin, Tiingo, World Bank)
- **Level 1B:** Sector + competitive positioning
- **Level 1C:** Technical + momentum signals (RSI, MACD, Bollinger, options flow)
- **Level 2:** Risk assessment + scenario modeling
- **Level 3:** Timing engine (entry window, catalyst calendar, volume confirmation)
- **Level 4:** Final synthesis → DecisionStrip generation
- **Constraint:** Pipeline runs in sequence. Do not skip levels.

### DecisionStrip
- **Purpose:** Final structured decision output for a specific asset
- **Output fields:** STATE / ACTION / WHY / CYCLE / TIMING / RISK
- **Constraint:** This is the ONLY place where actionable decisions appear.
- **Constraint:** DecisionStrip is generated by the pipeline, not by Radar or CandidatePool.

### CycleEngine
- **Purpose:** Macro + industry cycle context layer
- **Output:** Current macro stage, sector rotation, market style (risk-on/off), WHY (3 layers), risk warnings
- **Constraint:** Context layer only. Does NOT generate buy/sell signals.
- **Cache:** 4-hour TTL. Use `forceRefresh` only when macro conditions change materially.

### TimingEngine *(integrated in Level 3)*
- **Purpose:** Identify optimal entry windows based on catalyst calendar, technical confluence, volume confirmation
- **Constraint:** Timing confirms a decision; it does not create one.

## SECTION 4 — HARD RULES

These rules are absolute. No exceptions.

```
NEVER output BUY directly in OpportunityRadar
NEVER output BUY directly in CandidatePool
NEVER output BUY directly in CycleEngine

SELECT ≠ BUY
WATCH  ≠ BUY
PROMOTED ≠ BUY

Decision only happens in DecisionStrip.
DecisionStrip is only generated after Level 1–4 pipeline completes.
```

**Candidate status semantics:**

| Status | Meaning | Action Allowed |
|---|---|---|
| SELECT | Newly discovered, worth examining | Add to pool |
| WATCH | Waiting for timing or catalyst | Monitor only |
| PROMOTED | Entered full research pipeline | Analysis running |
| PASS | Rejected, not worth pursuing | Remove or archive |

**UI labeling rule:** Every module that is NOT DecisionStrip must display:
> `候选机会 · 进入 SELECT 阶段 · 非买入建议`
> or equivalent disclaimer in Chinese.

| **Status badges** | Every candidate must show lifecycle status badge |

These rules apply to ALL AI agents making code changes.

```
DO NOT redesign the system
DO NOT break the pipeline
DO NOT merge UI layers that are semantically separate
DO NOT add BUY signals outside DecisionStrip
DO NOT skip levels in the analysis pipeline
DO NOT rewrite working components — extend them

ALWAYS extend, never rewrite
ALWAYS maintain separation of layers
ALWAYS preserve existing tRPC procedure signatures
ALWAYS run TSC check before checkpoint
ALWAYS run vitest before checkpoint
ALWAYS output EXECUTION_FEEDBACK_PROTOCOL after task completion
```

**File ownership:**

| File | Owner | Rule |
|---|---|---|
